# equals方法

. ==是比较运算符

  1. ==既可以判断基本类型，也可以判断引用类型
  2. 判断基本类型是判断数值是否相同
  3. 判断引用类型是判断地址是否相同

. equals

  1. 只能判断引用类型
  2. 判断地址是否相同，在子类中会重写该方法，例如String，Integer等等都会重写该方法，我们在创建一个类时也会重写该方法
package study.com.object;

public class test {
    public static void main(String[] args) {
        String n1 = new String("123");
        String n2 = new String("123");
        System.out.println(n1.equals(n2));
        String n3 = new String("12");
        System.out.println(n1.equals(n3));
        Integer n4 = new Integer(123);
        System.out.println(n4.equals(n1));
    }
}

# hashCode

返回对象的哈希码值。 支持这种方法是为了散列表，如HashMap提供的那样 。

  1. 两个引用的对象，那么返回的哈希值是一样的，否则是不一样的
  2. hashCode在一些情况也是需要被重写的。

        System.out.println(n1.hashCode());

        System.out.println(n3.hashCode());

}

# finalize方法

当垃圾收集确定不再有对该对象的引用时，垃圾收集器在对象上调用该对象。 一个子类覆盖了处理系统资源或执行其他清理的finalize方法。

类似C++的 析构函数，会在对象被回收是进行调用。

什么时候被回收，是当该对象没有被引用时，jvm虚拟机会回收，再回收前会调用该方法。

垃圾回收机制是由系统来决定的，我们也可以使用System.gc()来进行手动回收。

package study.com.object;
import java.util.Objects;
class  Employee
{
    private String name;
    private int wage;
    Employee()
    {

    }
    public Employee(String name, int wage) {
        this.name = name;
        this.wage = wage;
    }

    @Override
    protected void finalize() throws Throwable {
        System.out.println("对象回收中......");
    }
}
class test1 {
    public static void main(String[] args) {
        Employee employee = new Employee();
        employee = null;
        System.gc();
    }
}

OUTPUT
对象回收中......

# 1. 多态

# 向上转型

有两个类，一个为A类，一个为B类，继承A类

public class A {
    public String name;

    public void show()
    {
        System.out.println("A.....");
    }
    public void cal()
    {
        System.out.println("cal.....");
    }

}

public class B extends A{
    public void show() {
        System.out.println("B....");
    }
    public void calB()
    {
        System.out.println("calB.....");
    }

}

我们创建一个A类并且指向B类A a = new B(),现在可以在使用a来访问B类在A类中出现的方法,但是不允许使用B类的特有方法。

总结，当出现向上转型的时候，我们需要看具体代码，前面一半为编译类型，后面为运行类型，运行方法时我们需要看的是运行类型，如果在运行类型中有我们需要的方法，那么我们就使用后面一个类型的，例如B类型，如果没有我们就使用A类型的。

# 向下转型

如果我们想在向上转型的类型中访问B中的特有方法是，我们就可以使用向下转型了，例如B bb = (B)a。

这样就可以使用B类的特有方法，但是注意，如果前面的A类型已经指向了除B类之外的类之后，那么我们就不可以让A向下转型为B类。

# 动态绑定机制

当我们使用向上转型时，我们使用方法一定要先到子类查看是否有该方法，无则调用父类的方法。但是对于属性来说，是先看编译类型里面的属性，而不是运行类型

# 2. 单例模式

1.所谓单例设计模式，就是采取一定的方法整个软件系统内，某个类只能有一个对象实例，并且只提供一个获取对象实例的方法 2.分为懒汉式与饿汉式 3.构造实现都是差不多的

  1. 将构造器私有化
  2. 类内构造对象
  3. 提供一个公共方法获取对象

# 饿汉式

在类加载的时候就进行构造对象，也就是说如果没使用该类，这个对象已经被构造出来了。

class GirlFriend{
    private String name;
    public static int n = 10;
    //实例化对象
    private static GirlFriend girl= new GirlFriend("德丽莎");

    private GirlFriend(String name)
    {
        System.out.println("饿汉式......");
        this.name = name;
    }

    public static GirlFriend getGirl()
    {
        return girl;
    }

    @Override
    public String toString() {
        return "GirlFriend{" +
                "name='" + name + '\'' +
                '}';
    }
}

# 懒汉式

如果未使用类提供的方法，那么该对象没有被构造。

class BoyFriend{
    private String name;
    public static int n = 10;
    private static BoyFriend boy;

    private BoyFriend(String name)
    {
        this.name = name;
    }

    public static BoyFriend getInstance()
    {
        System.out.println("懒汉式.......");
        //判断是否构造新对象
        if(boy ==null)
            boy = new BoyFriend("希尔");
        return boy;
    }

    @Override
    public String toString() {
        return "BoyFriend{" +
                "name='" + name + '\'' +
                '}';
    }
}

# 总结

  1. 懒汉式实在使用时创建对象，而饿汉式是在类加载创建对象
  2. 懒汉式存在线程问题
  3. 饿汉式会有资源浪费的可能

# 3. final使用

  1. final修饰为常量，无法改变
  2. 被final修饰的方法无法在子类中被重写
  3. final和static可以搭配一起使用，效率更高

# 4. 抽象类

父类的方法需要声明，但是又不知道具体写什么对象，可以使用abstract修饰该方法，即可，拥有抽象方法的类必定是抽象类，但是抽象类中也可以没有抽象方法。

abstract只能修饰类和方法，其余是不行的

# 5.抽象模板设计模式

建立一个工作，统计工作使用的时间，我们可以使用抽象类将工作放入该一个方法之中，使用abstract对工作进行抽象，因为不知道要进行什么工作，所以子类在继承该类后，实现此抽象工作方法即可。

package study.com.abstract_;

public abstract class currTime {
    public abstract void jod();
    public void curr()
    {
        long sta = System.currentTimeMillis();
        jod();

        long end = System.currentTimeMillis();

        System.out.println("时间: " + (end - sta));
    }
}

package study.com.abstract_;

public class Work extends currTime{
    @Override
    public void jod() {
        int t = 0;
        for(int i = 0; i < 100000; i++)
            t++;

    }
}

class A{
    public static void main(String[] args) {
        Work work = new Work();
        work.curr();
    }
}
